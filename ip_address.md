## IP Address: Understanding How it Works and its Limitations

An IP address is a unique numerical identifier assigned to a device that is connected to the internet or any other network. It is not meant to be unique to a user or a particular computer but rather identifies a Network Interface Card (NIC) in the device. This means that one device can have multiple NICs, and therefore, have multiple IP addresses assigned to it.

Furthermore, an IP address changes based on the network a device is connected to. For instance, when you switch from your home Wi-Fi network to a public network, the router in the public network assigns you a new IP address. This is facilitated by Dynamic Host Configuration Protocol (DHCP), which is responsible for temporarily assigning an IP address to a device. This process ensures that Wi-Fi networks do not run out of IPs as people come and go.

It is essential to note that IP addresses are not designed to be a unique identifier of a particular computer or user. Instead, they are intended to identify the device that is connected to a network. While an IP address can help track a device's activity, it does not provide comprehensive information on the user's identity or location.

The MAC address is a hardware identifier that is supposed to be unique to a NIC. However, it has its limitations. For instance, if a computer has a NIC that can be swapped out, changing the NIC will alter the MAC address. Additionally, computers can spoof their MAC addresses. For security reasons, smartphones often randomly generate a MAC address that they can change on the fly when they connect to a network.

It is also essential to note that MAC addresses typically do not make it past a local network. A web server does not see the MAC address of a computer it is connected to; it only sees the source IP and port number. Therefore, while the MAC address is a unique hardware identifier, it is not a reliable method of identifying a device on a global network.

Using an IP address to identify machines works well enough for most situations, but it is not a perfect solution to the identification problem. For instance, a software that identifies a device by IP will break immediately if the IP address changes. This change can happen for various reasons, such as booting up the computer or moving from one network to another.

If you want to identify a specific computer reliably, you would need something like the serial number from the CPU. The MAC address on a NIC can be overwritten by software, and IP packets can be routed to different systems transparently. Additionally, depending on how reliable you want the identification to be, there are more factors that you need to consider. For instance, a system can be connected to multiple networks at the same time.

In conclusion, IP addresses are an essential part of our internet-connected world, but they have their limitations. They are not unique identifiers for users or computers but rather identify a device that is connected to a network. While an IP address can help track a device's activity, it does not provide comprehensive information on the user's identity or location. Therefore, it is crucial to understand the limitations of IP addresses when using them for identification or tracking purposes.
